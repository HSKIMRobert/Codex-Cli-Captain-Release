# v0.0.9-pre

> Pre-release note: `v0.0.9-pre` keeps the captain-first CCC shape and tightens a few runtime and release-facing surfaces around generated config cleanup, docs routing, and long-session status guidance.

## What Changed

- Fresh generated `ccc-config.toml` files stay smaller by omitting internal routing, tool-routing, and runtime policy sections unless the operator has customized them.
- Runtime routing now falls back to the generated default router when a config has no `routing` section, so documentation and translation requests can route to `scribe`.
- Explicit disabled or custom routing remains respected when the operator includes a `routing` section.
- Minimal-config backfill keeps wholly omitted operational sections omitted, while existing operational sections can still receive missing generated defaults.
- Child-agent drift checks now compare any observed specialist `child_agent_id` with the expected owner; direct `captain` bypasses keep their separate acceptance gate.
- `scribe`, `companion_reader`, and `companion_operator` now default to medium reasoning/variant in the 0.0.9 release work, matching the README-facing role/default guidance.
- Captain and Way now generate task-specific expertise framing for specialist prompts so each subagent sees its role, stance, and thinking mode before it starts.
- Release docs now state that Codex CLI `/plan` and `/goal` are TUI slash commands, not CCC-native subagent commands, and that CCC maps them onto captain-owned Way/task-card state instead of claiming native slash-command integration.
- Long-session mitigation appears in status and compact status when context, token, or host-subagent pressure is high. CCC emits checkpoint and resume guidance, while `/compact`, `/new`, and `/exit` remain operator choices.
- Release-facing docs now point at the `0.0.9-pre` source version and describe the smaller generated config shape.
