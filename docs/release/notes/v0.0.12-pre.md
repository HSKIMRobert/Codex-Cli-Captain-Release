# v0.0.12-pre

> Pre-release note: `v0.0.12-pre` hardens the CCC Sisyphus runtime shape that `0.0.11-pre` introduced. It keeps `$cap` as the public entrypoint while improving structured flow, route evidence, app/CLI visibility, document-root graph and memory behavior, and skill-manifest-assisted routing/risk checks.

## What Changed

- Added phase-aware route validation so `PLAN_SEQUENCE` Way/tactician planning is not treated as drift just because the task is read-only.
- Added scheduler-owned planned-row materialization and persisted transition artifacts so LongWay rows, task cards, and status output stay aligned.
- Added target workspace resolution for structured mentions, document paths, non-git working directories, and ambiguous roots. CCC can now use the working document or repo root as the graph/memory target instead of assuming the host shell cwd is the project.
- Added document-root graph fallback and optional Tolaria graph/memory namespace mirroring under repo-like paths when Tolaria is installed.
- Added optional SSL skill manifests as machine-readable routing and risk evidence. `SKILL.md` remains the human instruction source, persisted run state and fan-in remain runtime truth, and missing or invalid manifests are non-blocking.
- Added compact Codex App/CLI LongWay output with a percentage-style progress gauge, focused checklist, planned row agent/model/reasoning projection, and compact subagent lifecycle events.
- Added Sisyphus harness coverage for PLAN_SEQUENCE, approval, scheduler materialization, routing, subagent fan-in, app-panel artifacts, completion, terminal scheduler state, and restart handoff.

## Scope Notes

`v0.0.12-pre` does not replace the `$cap` skill contract and does not make SSL manifests mandatory. SSL manifests are advisory evidence for Router, Sentinel, Arbiter, and Way planning support. The canonical truth remains persisted lifecycle artifacts, task cards, approved LongWay rows, fan-in, and run state.

The next `0.0.13-pre` cycle should evaluate whether this SSL slice improved planned-row routing, assignment drift, risk explanations, and Way planning before promoting the manifest layer into a broader SSL-backed Skill Registry.
