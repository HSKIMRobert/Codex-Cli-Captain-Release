# v0.0.1

- resets the public product and release history to a fresh Rust-only `v0.0.1` baseline
- removes the tracked TypeScript and Node implementation surface from the source repository
- makes the Rust runtime responsible for MCP serving, setup, and check-install
- installs the public `$cap` skill from the Rust package surface instead of the old npm package surface
- fixes the delegated worker spawn path so CCC streams the bounded worker prompt through stdin using the current `codex exec - --json` contract instead of relying on the older argument-only path
- restores delegated worker result and token visibility when the worker returns a bounded answer, so `ccc status` and `ccc activity` surface `Tokens:` and the latest worker preview again
- treats warning-only or non-JSON raw-events as worker failure instead of falsely marking the worker as completed
- caches worker message preview and token usage into delegation artifacts so status/activity do less best-effort raw-event recovery work
- trims repeated delegated worker prompt boilerplate to reduce avoidable token spend on every spawned specialist call
- moves the shared CCC config surface to `ccc-config.toml` and keeps legacy config files as migration fallbacks
- makes `setup` create the canonical default `ccc-config.toml` on first install instead of leaving config generation incomplete
- applies shared role model/profile/reasoning settings to delegated `codex exec` launches instead of ignoring them at runtime
- removes the ordinary manual `codex_bin` requirement by resolving the effective Codex binary from the run or local environment
- restores richer Rust status/activity visibility for fan-in readiness, active delegations, and best-effort token usage from worker raw events
- demotes `ccc_auto_entry` from the ordinary public `$cap` front door and makes explicit captain-first entry the canonical `0.0.1` path
- restores bounded worker reclaim by killing stalled delegated processes and reopening the run for captain follow-up instead of forcing immediate host fallback
- adds explicit captain reassignment inside the same run so captain can select the next specialist, retry the current specialist, or close the run with a resolved outcome
- renames the public product to `Codex-Cli-Captain` and the install surface to `Codex-Cli-Captain-Release`
- renames the public planning terminology to `Way` and `LongWay`
- rebuilds the install surface around release bundles from `Codex-Cli-Captain-Release`
