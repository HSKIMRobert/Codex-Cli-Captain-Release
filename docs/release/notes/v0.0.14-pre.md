# v0.0.14-pre

> Pre-release note: `v0.0.14-pre` tightens CCC into an intent-state-machine release. `$cap` remains the public entrypoint; persisted CCC run state, LongWay, task cards, checklist, fan-in, recovery, and status remain runtime truth.

## What Changed

- Sprint 1: added a compact `state_contract` and active-gate surface so status, checklist, and projection can name the current state, the required artifact, and the next allowed transition.
- Sprint 2: enforced subagent-first routing for specialist-owned work. Configured `ccc_*` agents are the required path while available; generic worker or explorer routes are rejected when they would bypass CCC-owned routing.
- Sprint 3: enforced evidence-before-mutation. Mutation-capable dispatch now blocks unless the run has persisted evidence or approved LongWay/task-card scope.
- Sprint 4: made recovery visibility explicit. `recovery_lane` now distinguishes fallback, reclaim, retry, and reassign state from the primary lane, and status/app-panel output keeps that separation visible.
- Sprint 5: added canonical `post_fan_in_captain_decision` truth so report, retry, reassign, recovery, and advance decisions come from one persisted envelope instead of implicit fan-in handling.
- Sprint 6: expanded app-panel and compact smoke coverage so the richer status payload, active gate, `recovery_lane`, and `post_fan_in_captain_decision` stay aligned across `ccc_render_app_panel` and compact status views.

## Scope Notes

`v0.0.14-pre` is stricter than the prior pre-release train, but it is not the end of release validation. The docs and smoke coverage now describe the gate model clearly; full publication checks, restart checks, and any remaining runtime fixes still need to finish before the release can be treated as complete.

The release still keeps these boundaries:

- configured `ccc_*` agents stay the default path for specialist-owned work while available
- terminal fallback or reclaim is a recorded exception, not a normal path
- evidence must exist before mutation-capable dispatch
- persisted CCC truth remains the source of completion, not hidden host state
